# Прогнозирование погоды по префиксным суммам

Flask-приложение анализирует CSV-файл с ежедневной температурой за год. Проект демонстрирует применение префиксных сумм, линейного поиска, сортировки, бинарного дерева поиска, очереди и стека отмены преобразований.

## Возможности программы

Приложение выполняет следующие операции:

1. вычисляет среднюю температуру за каждый месяц через префиксные суммы;
2. находит самый теплый и самый холодный день линейным поиском;
3. сортирует месяцы по средней температуре;
4. строит бинарное дерево поиска по температуре и рекурсивно извлекает все даты, где температура выше заданного порога;
5. сглаживает температурный ряд методом скользящего среднего с окном 7 дней;
6. сохраняет предыдущее состояние температурного ряда в стек и позволяет отменить последнее сглаживание;
7. поддерживает демонстрационные CSV-файлы и загрузку пользовательского CSV-файла через веб-интерфейс.

## Требования

- Python 3.9.12

## Установка и запуск

Создайте виртуальное окружение:

```bash
python -m venv venv
```

Активируйте окружение в Windows:

```bash
venv\Scripts\activate
```

Активируйте окружение в Linux/macOS:

```bash
source venv/bin/activate
```

Установите зависимости:

```bash
pip install -r requirements.txt
```

Сгенерируйте демонстрационные CSV-файлы, если папка `data` пуста:

```bash
python scripts/generate_weather_csv.py
```

Запустите Flask-приложение:

```bash
python app.py
```

После запуска откройте в браузере адрес:

```text
http://127.0.0.1:5000
```

## Формат CSV-файла

Пользовательский CSV-файл должен содержать две колонки:

```csv
date,temperature
2025-01-01,-5.2
2025-01-02,-4.7
2025-01-03,-3.9
```

Требования к данным:

- `date` — дата в формате `YYYY-MM-DD`;
- `temperature` — температура в градусах Цельсия;
- кодировка файла — UTF-8;
- файл может содержать данные за полный год или за произвольный период, но для задания предпочтительно использовать годовой ряд.

## Использованные алгоритмы

### Префиксные суммы

Для списка температур строится массив `prefix`, где `prefix[i]` хранит сумму элементов на интервале от начала ряда до позиции `i`, не включая `i`. Сумма температур за месяц вычисляется за O(1) по формуле:

```text
sum(left, right) = prefix[right] - prefix[left]
```

### Линейный поиск

Самый теплый и самый холодный день находятся одним проходом по списку температур. Сложность алгоритма — O(n).

### Сортировка

После вычисления средних температур список месяцев сортируется по возрастанию средней температуры.

### Бинарное дерево поиска

Для поиска всех дней с температурой выше порога строится бинарное дерево поиска. Ключом служит температура, значением — список дат. Если одинаковая температура встречается несколько раз, все соответствующие даты сохраняются в одном узле.

### Очередь

Сглаживание выполняется методом скользящего среднего с окном 7 дней. Для хранения последних семи температур используется очередь `deque`.

### Стек

Перед сглаживанием текущий температурный ряд помещается в стек. При отмене последнее состояние извлекается из стека и восстанавливается.

## Основные файлы

- `app.py` — маршруты Flask и управление пользовательской сессией;
- `core/weather_analyzer.py` — алгоритмы анализа температурного ряда;
- `scripts/generate_weather_csv.py` — генератор демонстрационных CSV-файлов;
- `templates/` — HTML-шаблоны;

## Проверка работы

Для комплексной проверки можно использовать три демонстрационных файла:

- `weather_moscow_2025.csv`;
- `weather_saint_petersburg_2025.csv`;
- `weather_krasnoyarsk_2025.csv`.

Файлы имеют разные параметры сезонности, амплитуды и случайного шума, поэтому результаты анализа заметно отличаются.

## Пример сценария проверки

1. Запустить приложение.
2. Выбрать `weather_moscow_2025.csv`.
3. Установить порог температуры, например `20`.
4. Нажать кнопку анализа.
5. Проверить таблицу средних температур по месяцам.
6. Проверить самый теплый и самый холодный день.
7. Проверить список дней выше порога.
8. Нажать кнопку сглаживания.
9. Убедиться, что первые строки ряда изменились.
10. Нажать кнопку отмены и проверить восстановление исходных значений.
